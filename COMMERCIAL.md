# Open-Core Model

vet is fully open-source software (Apache 2.0). Software supply chain verification — signing, verification, SBOM generation, and policy gating — is free, self-hostable, and will always remain open source.

vet has no commercial tier. It is a building block of the [Provabl](https://provabl.dev) suite; the commercial products are [attest Cloud](https://attest.provabl.dev) and [qualify Cloud](https://qualify.provabl.dev), which consume vet gate results as workload attributes in Cedar policy evaluation.

---

## What vet does (all open source)

| Component | Description |
|---|---|
| `vet sign` | Keyless cosign signing via Sigstore. Container images and local binaries |
| `vet verify` | Signature verification (cosign), SLSA provenance (gh CLI), CVE check (OSV API) |
| `vet sbom` | SBOM generation (syft) with optional cosign attestation |
| `vet gate` | Evaluates verification record against `.vet/policy.yaml`, writes Cedar workload attributes |
| `.vet/records/` | Per-artifact verification records (JSON) |
| `.vet/gate-result.json` | Cedar workload attributes: `context.workload.SLSALevel`, `Signed`, `CVECritical`, etc. |
| Policy evaluation | `min_slsa_level`, `cve_threshold`, `allowed_signing_ids` |

vet wraps `cosign`, `syft`, and `gh CLI` via `exec.Command` — no sigstore Go library dependency needed.

## How vet integrates with attest

attest's Cedar PDP reads `context.workload.*` attributes from `.vet/gate-result.json` during access evaluation. A researcher's environment can be gated on workload supply chain posture — not just their training completion.

```yaml
# Cedar policy example (generated by attest compile)
permit(principal, action, resource)
when {
  context.workload.SLSALevel >= 2 &&
  !context.workload.CVECritical &&
  context.workload.Signed
};
```

See `internal/gate/gate.go` for the exact Cedar attribute names written by `vet gate`.

## Suite relationship

```
vet sign image:tag              — sign workload artifact
vet verify image:tag            — verify signature + SLSA + CVEs
vet sbom image:tag --attest     — generate and attest SBOM
vet gate image:tag              — evaluate policy, write Cedar attributes
  ↓
attest scan                     — reads gate-result.json as workload context
```
