# Releasing vet

vet is the Provabl suite's **SLSA Level 3 pilot** (`provabl#5`). Tagging a
`v*.*.*` push triggers `.github/workflows/release.yml`, which produces, for
every release:

| Artifact | Produced by | Guarantee |
|---|---|---|
| `vet-<ver>-<os>-<arch>` | the `build` job (pure-Go cross-compile) | the binary |
| `…​.sig` + `…​.pem` | `cosign sign-blob --yes` (keyless) | who built it (Sigstore identity, logged to Rekor) |
| `*.intoto.jsonl` | `slsa-framework/slsa-github-generator` | **SLSA L3 provenance** — how it was built, on a non-falsifiable builder |
| `vet-<ver>.spdx.json` | `anchore/sbom-action` + `actions/attest-sbom` | what's inside it (SBOM) |

## Why this is L3 (not L2)

The previous workflow used `actions/attest-build-provenance` — **SLSA L2**: the
provenance is generated in the same job that builds the binary, so a compromised
build step could forge it. L3 requires the provenance to be generated by an
**isolated, trusted builder** the build job cannot tamper with. The
`generator_generic_slsa3.yml` reusable workflow runs in its own job
(`provenance:`), takes only the artifact **hashes** from the build job as input,
and signs the provenance itself — the build job never sees the signing key. That
isolation is the L3 guarantee.

The build job emits `outputs.hashes` (base64 `sha256sum` of the binaries); the
`provenance` job consumes it as `base64-subjects`. This is why the build matrix
was collapsed to a single runner — the generator takes one combined hashes blob,
and vet is `CGO_ENABLED=0` pure Go, so all four targets cross-compile from one
ubuntu runner.

## Cutting a release

```bash
# 1. Land all changes on main; ensure CHANGELOG [Unreleased] is filled.
# 2. Promote [Unreleased] → [X.Y.Z] - <date> in CHANGELOG.md, set main.version.
# 3. Tag and push:
git tag -s vX.Y.Z -m "vX.Y.Z"
git push origin vX.Y.Z
```

## Verifying a release (consumers)

### Verify SLSA L3 provenance with `slsa-verifier`

The generic generator publishes **one** provenance file, `multiple.intoto.jsonl`, that attests all
the release binaries together (not one `.intoto.jsonl` per binary).

```bash
# Install: go install github.com/slsa-framework/slsa-verifier/v2/cli/slsa-verifier@latest
slsa-verifier verify-artifact \
  vet-vX.Y.Z-linux-amd64 \
  --provenance-path multiple.intoto.jsonl \
  --source-uri github.com/provabl/vet \
  --source-tag vX.Y.Z
# → PASSED: SLSA verification passed
```

`slsa-verifier` confirms the provenance was produced by the trusted
slsa-github-generator builder (the L3 builder identity), that the artifact hash
matches, and that the source repo/tag are as expected. Verified on v0.2.0:

```
Verified build using builder ".../generator_generic_slsa3.yml@refs/tags/v2.1.0" at commit <sha>
Verifying artifact vet-v0.2.0-darwin-arm64: PASSED
PASSED: SLSA verification passed
```

> **Not `gh attestation verify`.** The generic generator uploads provenance as a **release asset**
> (`multiple.intoto.jsonl`), not to GitHub's attestations API — so `gh attestation verify … --repo
> provabl/vet` returns HTTP 404. `slsa-verifier verify-artifact` (above) is the correct consumer
> check.

### Verify the cosign signature

```bash
cosign verify-blob \
  --signature   vet-vX.Y.Z-linux-amd64.sig \
  --certificate vet-vX.Y.Z-linux-amd64.pem \
  --certificate-identity-regexp '^https://github.com/provabl/vet/' \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  vet-vX.Y.Z-linux-amd64
```

## Dogfooding

vet itself verifies provenance — once a release is published, vet can verify its
own artifacts:

```bash
vet verify github.com/provabl/vet --source github.com/provabl/vet --min-slsa-level 3
```

This exercises vet's own structural SLSA-level parser (it should report **L3**
for releases built by the generator, vs L2 for the old workflow), closing the
loop between the tool and the supply-chain guarantees it checks.

## Rollout note (provabl#5)

vet is the **pilot**. The real L3 proof is produced on the next tagged release —
it cannot be exercised in a PR. Once a `vet vX.Y.Z` release verifies clean with
`slsa-verifier` (PASSED, builder = slsa-github-generator), the same
`build → provenance (generator_generic_slsa3) → sbom` pattern rolls out to the
other suite repos (attest, ground, qualify, nitro, evidence).
